Tingginya redudansi data pasien menunjukkan bahwa data master (pasien) tersimpan berulang pada data kunjungan. Hal ini tentunya menyebabkan pemborosan penyimpanan, meningkatkan risiko inkonsistensi saat pembaruan data (misalnya perubahan alamat/tipe pasien), dan sulit menjamin integritas data jika tetap disimpan dalam bentuk CSV. Oleh karena itu, diperlukan database relasional dengan pemisah tabel master (pasien, dokter, klinik) dan tabel transaksi (visit) untuk normalisasi dan konsistensi.

  1. Relasi Pasien -> Kunjungan

    • Setiap pasien minimal memiliki 1 kunjungan, dan sebagian besar pasien melakukan lebih dari satu kunjungan.

    • Nilai median (3) dan Q3 (5) menunjukkan bahwa mayoritas pasien melakukan kunjungan berulang.

    • Adanya pasien dengan hingga 28 kunjungan mengindikasikan relasi satu pasien ke banyak kunjungan (one-to-many).

    Pasien merupakan entitas master yang direferensikan oleh banyak data kunjungan. Menyimpan seluruh informasi pasien berulang pada setiap baris kunjungan menyebabkan redundansi tinggi, sehingga secara konseptual data ini harus dipisahkan menjadi tabel patient dan tabel visit yang dihubungkan melalui foreign key.

  2. Relasi Dokter -> Kunjungan

    • Setiap dokter melayani banyak kunjungan pasien, bahkan dokter dengan jumlah kunjungan paling sedikit pun tetap memiliki lebih dari satu kunjungan.

    • Median dan rata-rata yang tinggi menunjukkan bahwa dokter adalah entitas yang berulang secara masif pada data kunjungan.

    • Nilai maksimum (2.083 kunjungan) memperkuat bahwa satu dokter dapat terkait dengan ribuan transaksi kunjungan.

    Dokter merupakan entitas master dengan relasi one-to-many terhadap kunjungan. Oleh karena itu, informasi dokter seharusnya disimpan dalam tabel tersendiri dan direferensikan melalui foreign key pada tabel kunjungan, sehingga dibutuhkan database relasional untuk mengatasi hal ini

5. Cek “1NF violation”

# deteksi kolom yang mengandung pola JSON array: "[" ... "]"
is_repeating_group <- function(x) {
  x <- x[!is.na(x)]
  if (length(x) == 0) return(FALSE)
  x <- x[1:min(2000, length(x))]
  any(str_detect(as.character(x), "^\\["))
}

kolom_repeating <- names(df)[map_lgl(df, is_repeating_group)]

# ukur "berapa item per sel" untuk kolom repeating (estimasi cepat berbasis jumlah koma)
count_items_rough <- function(x) {
  x <- as.character(x)
  x[is.na(x)] <- "[]"
  x <- str_trim(x)

  # [] -> 0
  out <- ifelse(x == "[]", 0L, 1L + str_count(x, ","))
  out
}

if (length(kolom_repeating) > 0) {
  ringkas_repeating <- map_dfr(kolom_repeating, function(k) {
    v <- df[[k]]
    len <- count_items_rough(v)
    tibble(
      kolom = k,
      mean_item = mean(len, na.rm = TRUE),
      max_item  = max(len, na.rm = TRUE),
      prop_gt1  = mean(len > 1, na.rm = TRUE)
    )
  })
  ringkas_repeating
}

1NF (First Normal Form) merupakan aturan paling dasar dalam normalisasi database. 1NF mensyaratkan bahwa setiap atribut dalam tabel tidak mengandung kelompok berulang (repeating group), serta setiap record dapat diidentifikasi secara unik melalui primary key.

Deteksi kolom “repeating group” (nilai berbentuk list/array [...]) menunjukkan adanya banyak item (misal diagnosis/tindakan/obat) dalam satu sel yang melanggar prinsip 1NF. Ini merupakan bukti kuat perlunya pemecahan ke tabel detail (line-item) yang dihubungkan dengan foreign key.

Dalam 1NF, setiap sel harus berisi 1 nilai (atomik).
Jika ada baris yang menyimpan lebih dari 1 item (misal obat lebih dari satu per kunjungan), maka itu melanggar 1NFdan harus dipindahkan ke tabel detail.

  1. Diagnosis

  • mean_item = 1.12

  • max_item = 2

  • prop_gt1 = 0.1207 (≈ 12,07% baris punya >1 diagnosis)

    Ada sebagian kunjungan yang memiliki lebih dari satu diagnosis hal ini menandakan indikasi repeating group.

  1. Treatment & Treatment_fee

  • mean_item = 0.66

  • max_item = 2

  • prop_gt1 = 0.0313 (≈ 3,13% baris punya >1 tindakan)

Walaupun proporsinya kecil, tetap ada kunjungan dengan >1 tindakan.
Artinya treatment bukan selalu 1 nilai per kunjungan hal ini berpotensi melanggar 1NF.

  1. semua kolom obat

  • mean_item = 1.893

  • max_item = 3

  • prop_gt1 = 0.7382 (≈ 73,82% baris punya >1 obat)
    Mayoritas kunjungan (hampir 3 dari 4 kunjungan) berisi lebih dari satu obat.

Berdasarkan hasil pengukuran prop_gt1, dataset tidak memenuhi 1NF untuk atribut diagnosis, tindakan, dan terutama obat, karena terdapat banyak baris yang menyimpan lebih dari satu nilai dalam satu kolom (non-atomik / repeating group).
